Right - Responsive Admin Template

Right is pretty simple and light Bootstrap template that will help you to build the admin panel for your project.

Version: 0.3.0

Documentation

Next is a brief description of the key points that can be useful to you in the process of working with the template.

Overview

Right is developed on the principles of combining beautiful and functional design, but also includes advanced tools to build frontend. An important role is assigned to the speed of development and ease future code maintenance. Therefore, Right uses such tools as Jade and Less for html templating and css-preprocessing. All source code is building automatically via Gulp. Together, they greatly simplify and speed up the development process. But don't worry if you have never worked with these tools. Together with the sources in the folder is fully assembled and ready to go html template that you can immediately implement in your project.

Layout structure

All pages of the template (except login.html) consist of the following parts:

  1. .wrapper - main container for all elements
  2. .header-navbar - top navigation panel
  3. .dashboard - sidebar and content container
  4. .sidebar - side navigation panel
  5. .main - common content container
<div class="wrapper">
    <nav class="header-navbar"></nav>
    <div class="dashboard">
        <div class="sidebar"></div>
        <div class="main"></div>
    </div>
</div>

Responsivity

When the screen width <= 768px, the template adapts. .sidebar is displaced off the screen and appears when you click the menu button. .topnavbar is replaced with .header-navbar-mobile element which contains the mobile menu button and the page title.

Pages

The template includes 8 functional pages:

  1. index.html - page with brief statistics and some informational widgets.
  2. products.html - the page can be used to manage the list of products or goods.
  3. orders.html - orders page.
  4. users.html - users or customers manage page.
  5. login.html - login page.
  6. blank.html - blank page, is used to create new functional pages.
  7. inbox.html - mails list page.
  8. compose.html - new mail form.

Also in the template included demo pages that show the individual components of the interface.

Building the project

All the following instructions assume you are using the Windows operating system. For other OS the accuracy of the above commands is not guaranteed, but you can find similar solutions for your system on the official websites of the described tools.

Getting started

To start to build the project from source, you need to install Node.js. All you need for this is on nodejs.org.

After Node.js will be installed, navigate to the project folder, which contains your package.json file. Now we need to get the development dependencies. Type the following command in OS command line:

npm install

This command will load the Gulp and all of its modules needed for further work.

If you get any error messages, make sure that you have the Node.js and npm installed. And check the package.json file in the startup directory. Also, you can try to run the command as administrator.

Gulp

Gulp.js is the automated build system that we will use for generating html from jade and css from less. It also allows you to automate operations such as resource minification, files concatenation and more. There all of necessary tasks are in the gulpfile.js.

To begin building your project just write in the command line:

gulp build

This will start the builder that will process all the files in the ./source folder and will store the result in ./dist.

The command gulp watch will track all the changes in the folder ./source real-time and automatically update the result in ./dist.

To consistently run the task build and watch you can run gulp or gulp default.

gulp clean deletes the directory ./dist.

Jade

Jade is a powerful template engine, which allows to considerably speed up and simplify the elaboration of html projects. Read more about its capabilities and syntax on the official website.

The structure of jade templates:

Right/
  ├ source/                              source files
  ├───jade/                              jade folder
  ├──── include/                         static content files
  ├──── parts/                           common parts of template
  ├────── layout.jade                    common layout
  ├────── main.jade                      main block
  ├────── navbar.jade                    top navigation
  ├────── sidebar.jade                   side navigation
  ├──── blank.jade                        all end pages
         .
         .
         all *.jade pages

Any final page begins with the inheritance common template layout.jade , which in its general form looks like this:

doctype html
html(lang="en")
    head
        meta(charset='utf-8')
        meta(http-equiv='X-UA-Compatible', content='IE=edge')
        meta(name='viewport', content='width=device-width, initial-scale=1')
        title Right - Bootstrap Admin Template
        block css
        link(href='css/right.css', rel='stylesheet')
    block vars
    body
        .wrapper
            include ./navbar.jade
            .dashboard
                include ./sidebar.jade
                include ./main.jade
        script(src='libs/jquery/jquery.min.js')
        script(src='libs/bootstrap/js/bootstrap.min.js')
        block js
        script(src='js/main.js')

Each page is constructed on the basis of this template, including the individual parts using the operator include, replacing the special tags block with they own content.

Any final page template looks like (for example, blank.jade ):

extends ./parts/layout.jade
block vars
    - page = 'Blank';
block main
    .main-heading
        .main-title
            ol.breadcrumb
                li.active Blank
        .main-filter
            form.main-filter__search
                .input-group
                    input.form-control(type="text", placeholder="Search...")
                    span.input-group-btn
                        button.btn.btn-default(type="button")
                            .fa.fa-search
    .container-fluid.half-padding
        .template.template__blank
            .row
                .col-md-12
                    .panel.panel-danger
                        .panel-heading
                            h3.panel-title Blank Page
                        .panel-body
                            p This is a blank page.

Less

Less is a CSS pre-processor, meaning that it extends the CSS language, adding features that allow variables, mixins, functions and many other techniques that allow you to make CSS that is more maintainable, themable and extendable.

The structure of less directory:

Right/
  ├ source/                              source files
  ├───less/                              less folder
  ├──── dark/                            color theme container folder
  ├────── blocks/                        layout structure elements
  ├────── bootstrap/                     customized bootstrap styles
  ├────── libs/                          customized third-party libs
  ├────── colors.less                    colors
  ├────── mixins.less                    less mixins
  ├──── right.dark.less                  color theme container file

Color Themes

Version 0.3.0 includes 3 color schemes: dark, lilac and light. The styles for each of them are located in separate files. Thus, to use a specific color scheme, you need to connect to the page a certain stylesheet.

Right/
  ├ dist/
  ├─── css/
  ├───── right.dark.css             Dark theme
  ├───── right.light.css            Light theme
  ├───── right.lilac.css            Lilac theme
